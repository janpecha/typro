Specifikace RSS 2.0

Jedná se o neúplný překlad textu RSS 2.0 Specification (RSS 2.0 at Harvard Law) .

Poznámka: elementy, které musí být uvedeny, jsou zvýrazněny tučně.

Definice XML

Všechny RSS dokumenty musí být v souladu s  XML 1.0 . V definici XML je vhodné uvést atribut encoding . 

<?xml version="1.0" encoding="utf-8"?>

Element RSS

Za definicí XML následuje element <rss> , který obsahuje vlastní položky RSS kanálu. Element musí obsahovat atribut version , ten musí mít hodnotu 2.0 . 

<rss version="2.0">

Element Channel

Element channel je jediným potomkem elementu <rss> ( zdroj? ). Obsahuje metadata a vlastní položky (články, komentáře, …). Element obsahuje následující potomky:

Povinné elementy

title
jméno kanálu (webu)
Příklad: GoUpstate.com News Headlines
link
url na webovou stránku (mělo by korespondovat s kanálem)
Příklad: http://www.goupstate.com/
description
popis kanálu (fráze, věta)
Příklad: The latest news from GoUpstate.com, a Spartanburg Herald-Journal Web site.

Volitelné elementy

language
jazyk, ve kterém je kanál psán
možné hodnoty: kódy jazyků , nebo definice kódů od W3C
Příklad: cs
Příklad: en-us
copyright
poznámka o autorských právech vztahujících se na obsah v kanálu
Příklad: Copyright 2002, Spartanburg Herald-Journal
managingEditor
emailová adresa osoby, která je zodpovědná za obsah v kanálu
Příklad: geo@ herald.com (George Matesky)
webMaster
emailová adresa osoby, která je zodpovědná za technické aspekty týkající se kanálu
Příklad: betty@ herald.com (Betty Guernsey)
pubDate
datum, kdy byl obsah v kanálu publikován (lze chápat jako datum kdy byl do kanálu přidán nový obsah ( zdroj? ))
více informací o formátu data a času v samostatném oddílu Datum a čas
Příklad: Sat, 07 Sep 2002 00:00:01 GMT
lastBuildDate
čas, kdy byl obsah kanálu naposledy změněn
Příklad: Sat, 07 Sep 2002 09:42:31 GMT
category
specifikuje jednu nebo více kategorií, které kanálu patří
řídí se stejnými pravidly jako element category v elementu  item
Příklad: <category>New­spapers</cate­gory>

generator
řetězec označující program použitý pro generování kanálu
Příklad: MightyInHouse Content System v2.3
docs
URL, která odkazuje na dokumentaci formátu použitého v souboru RSS
Příklad: http://blogs.law.harvard.edu/tech/rss
cloud

více info zde [en]
Příklad:

ttl
počet minut, které udávají, jak dlouho může být kanál kešován před obnovou ze serveru
Příklad: <ttl>60</ttl>
rating
PICS rating kanálu

specifikuje kolik hodin může agregátor vynechat kontrolu kanálu
skipDays

Element channel obsahuje ještě následující potomky:

Element Image

Volitelný. Specifikuje GIF, JPEG nebo PNG obrázek, který může být zobrazen společně s kanálem.

Obsahuje následující povinné elementy:

url
URL obrázku
title
popis obrázku
při zobrazení v HTML dokumentu měl hodnotu přebrat atribut alt elementu img .
link
URL stránky, která se otevře po kliknutí na obrázek ( ok? )
description
( určitě povinný? )
text, který by měl přebrat atribut title odkazu, který obaluje obrázek ve formátu HTML

Dále následující ne povinné elementy:

width
číslo, šířka obrázku v pixelech
maximum je 144
výchozí hodnota 88
height
číslo, výška obrázku v pixelech
maximum je 400
výchozí hodnota 31

Element Item

Definuje jednu položku RSS kanálu. Počet položek není omezen.

Jeden z následujích elementů musí být vždy uveden:

title
titulek položky
Příklad: Venice Film Festival Tries to Quit Sinking
description
anotace položky
Příklad: Some of the most heated chatter at the Venice Film Festival this week was about the way that the arrival of the stars at the Palazzo del Cinema was being staged.

Ostatní elementy jsou ne povinné:

link
URL položky
Příklad: http://nytimes.com/…/07FEST.html
author
e-mailová adresa autora položky
Příklad: lawyer@ boyer.net (Lawyer Boyer)
category
jedna nebo více kategorií
TODO: doplnit
comments
URL stránky s komentáři, které patří k dané položce
Příklad: http://www.myblog.org/…comments.cgi?…
enclosure
popis multimediálního objektu připojeného k položce
TODO: doplnit
guid
velice důležitý element
řetězec unikátně identifikující položku
lze uvést atribut isPermaLink nastavený na hodnotu true (výchozí hodnota), nebo  false
atribut isPermaLink specifikuje zda je hodnota elementu guid trvalým odkazem na položku
velice často URL položky (URL se nemění příliš často)
Příklad: http://inessential.com/2002/09/01.php#a2
pubDate
kdy byla položka publikována
viz Datum a čas
Příklad: Sun, 19 May 2002 15:21:36 GMT
source
RSS kanál, ze kterého položka pochází

Element TextInput

Definuje textové pole. Účel je záhadou. Můžete takto definovat vyhledávací políčko, nebo poskytnout čtenářům možnost zpětné vazby. Ve spoustě případů je tento element ignorován.

Obsahuje následující povinné elementy:

title
popisek odesílacího tlačítka
description
popis textového pole
name
název textového objektu v textovém poli
link
URL CGI skriptu

Datum a čas

Všechny zápisy data a času odpovídají specifikaci RFC 822 , s vyjímkou roku, který může být zapsán dvěma nebo čtyřmi číslicemi (upřednostněny jsou čtyři číslice).

Formát podle RFC 822  vypadá následovně: 

Mon, 15 Aug 05 15:52:01 +0000

Poznámka: Při generování data v PHP lze použít konstanty DateTime::RFC822 či DATE_RFC822 .
